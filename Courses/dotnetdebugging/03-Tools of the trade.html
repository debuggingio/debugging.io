windbg

Debugger installed with the debugging tools for windows.

WinDbg can be used for debugging kernel-mode memory dumps,user-mode crash dumps(This is known as post-mortem debugging) or debug live process.

Capturing hang dumps

  • using debugdiag
  • using task manager

using debugdiag

  • Launch debugDaig
  • Go to Processes tab
  • Find w3wp.exe process

Create Ful user Dump in debugdiag

  • Find w3wp.exe process
  • Right Click and Choose "Create Full UserDump"

Using task manager

  • Find w3wp.exe process
  • Right Click and Choose "Create Dump File"
