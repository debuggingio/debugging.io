.NET Core

  • .NET Core is a set of runtime, library and compiler components
  • Compiles your code with a JIT compiler or ahead of time with .NET Native.
  • .NET Core manages memory with a garbage collector
  • .NET Core runs on Windows,Linux, OS X and FreeBSD

What is .NET Core

.net core is one framework for all.Reduced overall footprint and runs across all OSes

has two major components

  • CoreCLR-a small runtime that is built from the same codebase as the .NET Framework CLR -Microsoft.CoreCLR NuGet package.
  • CoreFx -shipped as System.* NuGet packages on NuGet.org

DNX (.NET Execution Environment)

Applications can target DNX and run on all platforms

  • Bootstraps the hosting runtime(.net core)
  • Managed entry point discovery
  • Excutes the application

Why build DNX?

DNX was built for running cross-platform ASP.NET Web applications or cross-platform console apps

  • DNX simplifies cross-platform development using .NET Core
  • DNX provides tools for installing, creating and managing NuGet packages.
  • Easy to work with open source projects-DNX projects helps you easily replace an existing dependency with its source code ,then debug the source and modify it without changing your app.
